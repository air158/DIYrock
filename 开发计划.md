## 一、Demo 拆解:它到底是个什么交互

视频里的「珠了个珠 / CRYSTAL HEALING」是一个**水晶手串 DIY 设计器**(抖音小程序形态)。核心交互链路是:

- 中央画布是一个**圆环珠串**,珠子沿圆周首尾相接排布,中间是品牌 logo
- 顶部实时显示**手围(cm)+ 总价(¥)**,珠子不够时提示「长度过短」——这是即时校验
- 底部抽屉是选材区:`珠子/配饰` 两个 Tab,左侧分类竖排(白水晶/紫水晶/黄水晶 · 隔片/隔珠/花托),右侧是带图片+规格(8/10/12mm)+价格的卡片网格
- 点一颗珠子 → 它**入环**,整圈珠子重新均匀布局、圆半径随之变大,手围和总价同步跳动
- 底部三个动作:清空 / 保存 / 完成设计
- 结尾是「大家都在抖音 搜索:水晶diy小程序入口」的引流卡

它最值钱的部分有两个:**圆环自动布局引擎**(珠子相切、半径自适应、手围=物理周长)和**即时反馈闭环**(选→入环→数值跳动→零学习成本)。

## 二、赛道建议:主攻赛道三

我建议的产品定位:**「今日水晶 · 能量搭子」**

想投**赛道二**(内容重构):让用户贴一条水晶手串视频 → AI 解析出珠子配方与寓意 → 一键复刻可微调。这样「把内容变成可执行的工具」就成立。下面的开发计划两条路通用,差异只在入口模块。

## 三、技术选型(2 天优先级:快 > 全)

| 层 | 选择 | 理由 |
|---|---|---|
| 框架 | React + Vite + TS | 启动快,组件化适合抽屉/卡片 |
| 样式 | Tailwind + CSS 变量 | 主题色随主导水晶切换 |
| 动画 | Framer Motion | 入环 spring、FLIP 重排、数字滚动一把梭 |
| 状态 | Zustand | 珠串数组单一数据源,布局纯函数派生 |
| 渲染 | 绝对定位 div + transform(珠子)| 易动画、易点击,无需 Canvas |
| AI | openai支持| 字节系活动更对味;你本地 vLLM 仅适合开发,场内评委手机扫码访问不到内网 |
| 部署 | 国内静态托管(OSS+CDN / Cloudflare Pages)+ 二维码 | 现场扫码必须秒开、能访问 |

珠子**不用准备真实图片**,用 CSS `radial-gradient` 程序化渲染水晶球(球体渐变 + 高光点 + 投影),按水晶类型换色,既省素材又统一风格;有现成 PNG 再替换。

## 四、核心:圆环布局引擎(技术亮点,先把它打通)

整个产品成败在这一个函数。物理模型很简单,讲清楚就是加分项:

- 周长 `C = Σ(每颗珠子直径)`,即**手围**(mm 换 cm)
- 圆半径 `R = C / (2π)`
- 每颗珠子占的圆心角 `θ_i = (直径_i / C) × 2π`,沿累加角度摆放,珠子自然首尾相切
- 珠子坐标 `x = R·cos(角), y = R·sin(角)`,用 `transform: translate()` 定位

数据源只存一个数组 `beads: [{type, size, color}]`,位置全部由它纯函数派生。增删珠子 → 数组变 → 重新计算 → Framer Motion 的 `layout` 属性自动做 FLIP 重排动画。手围超阈值就弹「过短/过长」校验。这套做出来,赛道三的「体验完整闭环」基本拿满。

## 五、样式 / 视觉系统

- 整体走抖音小程序的**轻盈 + 疗愈高级感**:浅灰渐变背景、毛玻璃面板(`backdrop-blur`)、大量留白
- 背景做**呼吸式渐变**,色调随当前主导水晶色缓慢偏移(紫=平静、黄=活力、白=纯净)
- 珠子高光统一光源方向,投影柔和;选中珠子加描边光晕
- 底部抽屉:左分类竖排(选中态橙色竖条 + 高亮)、右卡片网格(图/名/规格/价),和原 Demo 一致
- 顶部状态条做成胶囊,手围/总价分块,校验文字用警示色

## 六、动画清单(按场景,这是评委的直观打动点)

- **入环**:新珠子从抽屉位置飞入 + 缩放弹出,随后全环 spring 重排、半径生长
- **删除**:珠子缩小淡出,缺口被相邻珠子补齐(同一套 FLIP)
- **数值**:手围/总价 count-up 滚动,跨阈值时手围数字闪一下红
- **闲置**:手串极缓慢自转,营造「活着」的质感
- **能量卡生成**:AI 出结果时走一段微粒/流光 shimmer,珠子逐颗点亮入环
- **完成设计**:手串高光闪烁 → 翻转成一张可分享海报卡(对应赛道三复访/传播)
- **入口转场**:首屏模拟抖音信息流卡片上滑出现,强化「刷到即用」

## 七、AI 集成

最小可用就三个能力,都让大模型返回**结构化 JSON**便于驱动 UI:

1. 意图→配方:输入「最近压力大想要平静」或日期/天气 → 返回 `{beads:[{type,size,count,reason}], theme, copy}`
2. 寓意解读:点某颗珠子,出一句话功效/寓意(给「价值感」)
3. 文案:生成每日能量卡的标题与一句话(给传播)

Key 放在轻量服务端代理里,别写进前端。开发期可指你本地 vLLM 跑通逻辑,**部署演示一定换公网模型**,否则评委手机扫码调不通。
